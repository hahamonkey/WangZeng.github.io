Hexo+Github搭建博客

留言,参与

03 04, 2019点击

一直打算重新整一个博客，由于实习和准备毕业论文的原因搁置下来。刚好利用这几天的闲余时间，也在想要用什么方式去实现，最终选择了Hexo。这里分享一下从0到1以及这个过程中遇到的一些问题，又是如何解决的。(注：本文基于Mac系统)

使用GitHub Pages + Hexo有以下几点好处：

  • 无需关注生成网页源码的细节，只需用markdown攥写你的文章
  • github免费提供无限资源空间
  • Hexo基于Node.js，其对数据的静态处理方式来的更高效
  • 相对于搭建独立博客，有绝对的控制权，管理方便

一、开始之前

I. 环境配置

  • 安装Node.js
  • 安装Git

II. 创建Github项目

  1. 打开github，新建项目，项目名称推荐username.github.io(其中username为你的github名) 的形式，原因后面会提到，另外勾上 README 初始化选项；
  2. 打开setting，找到Github Pages，choose a theme选择一个主题。重新进入设置页找到 Github Pages
  3. 点击 https://hahamonkey.github.io 就会出现相应的网页了

二、Hexo安装

I. 安装Hexo

首先，在合适的地方新建文件夹，例如在D盘新建Blog(D:\Blog)目录。接着打开控制台，定位到博客目录，键入下面的指令：

npm install hexo-cli -g  （可能会有几条报错语句，无视就好）
npm install hexo --save 
hexo -v (查看是否安装成功)

接着初始化博客hexo init，hexo g生成静态资源，hexo s打开本地服务器预览网站。到此，单独的两部分都完成了。

II. 本地代码远程连接Github

  1. 控制台输入：
git config --global user.name "username"   // username->github用户名
git config --global user.email "email"  // email->github绑定邮箱
  1. 生成密钥 SSH key:
ssh-keygen -t rsa -C "email"
eval "$(ssh-agent -s)"   (添加密钥到ssh-agent)
ssh-add ~/.ssh/id_rsa  (添加生成的SSH key到ssh-agent)
  1. 打开github，新建SSH。接着打开控制台终端定位到根目录，输入：
cd ~/.ssh
open .

复制id_rsa.pub文件中的内容到新建的SSH中

  1. 打开 Blog 根目录下的_config.yml文件，在文件的末尾添加如下配置：
deploy: 
  type: git
  repository: https://github.com/hahamonkey/hahamonkey.github.io
  branch: master

（注意： 冒号后跟空格符）

III. 写作

到了这一步，搭建hexo博客基本上算是完成了，接下来可以看一下搭建的博客是什么样子。记得每次新建文章之前 hexo clean 删除缓存文件，防止修改分类和标签或者修改主题之后无效，好了，那就来试一下吧：

hexo clean 
hexo new post filename    生成文件后，markdown语法编辑，具体参考 hexo官方文档
hexo g      生成静态文件
hexo s      打开本地服务器预览（选择性操作）
hexo d      部署到 Github Pages

接着打开 https://hahamonkey.github.io 查看效果

IV. 更换主题

你可能会觉得一开始选择的博客主题不太理想，好吧，当然会考虑到，也给你提供了一些选择。比较之下，这款BlueLake甚合我意，那就开始吧：

  1. 首先，下载这个主题
git clone https://github.com/chaooo/hexo-theme-BlueLake.git themes/bluelake
  1. 打开Blog根目录下的 _config.yml 文件，找到 theme 字段,设置 theme: bluelake
  2. 清除缓存 hexo clean，生成静态文件 hexo g，部署到 Github Pages hexo d(之后统一简称“重新部署”)

V. 绑定域名

在这之前，博客就算是已经完成了，但是没有一个专有的域名就会觉得档次跟不上。之前我们在 github 新建项目的时候强调要用自己的 github 用户名 开始，以.github.io 作为后缀，如果不是这样，当然可以，那我们打开博客的地址可能就是下面这样：

https://username.github.io/xxxx.github.io

很怪异不是吗？不仅是为了方便，也是防止不必要的小麻烦。下面就说如何绑定域名
